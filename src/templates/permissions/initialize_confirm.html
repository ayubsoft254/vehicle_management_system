Initialize Default Permissions

Set up role-based permissions with smart defaults

What This Does

This will create default permission sets for all 6 system roles across all 14 modules. The system will intelligently assign appropriate access levels based on each role's typical responsibilities.

Expected Result

Approximately 84 permission records will be created (6 roles × 14 modules = 84 combinations)

Default Permission Matrix

Admin

Full access to all modules with complete control

Manager

Full access to most modules, read-only access to audit and permissions

Sales

Specialized for customer-facing operations:

  • • Read/Write: Vehicles, Clients, Payments, Insurance, Documents
  • • Read-Only: Dashboard, Reports
  • • No Access: Payroll, Audit, Permissions (sensitive areas)

Accountant

Specialized for financial operations:

  • • Full Access: Payments, Payroll, Expenses, Reports
  • • Read-Only: Repossessions, Auctions
  • • No Access: Permissions (security restriction)

Auctioneer

Specialized for auction operations:

  • • Full Access: Auctions
  • • Read/Write: Repossessions
  • • No Access: Payroll, Permissions (restricted areas)

Clerk

Basic access for data entry:

  • • Read-Only: Dashboard, Vehicles, Clients, Documents
  • • No Access: All other modules (least privilege principle)

Important Notes

Existing permissions will NOT be overwritten - only missing ones will be created

This is safe to run multiple times - it only creates missing records

You can modify any permission after initialization through the normal edit process

All created permissions will be marked as active by default

The creation action will be logged for audit purposes

When to Use This

Initial Setup: When setting up the system for the first time

After Updates: When new modules are added to the system

Missing Permissions: To fill in any gaps in the permission matrix

Reset Baseline: To ensure all role-module combinations exist
